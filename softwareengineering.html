Software Engineering

Project Overview

I’m developing the SEC EDGAR API Project, a full-stack application designed to provide seamless access to the SEC's EDGAR (Electronic Data Gathering, Analysis, and Retrieval) system. This project is hosted on AWS and serves as a robust API service for managing Central Index Key (CIK) data and other SEC filings.

The SEC (Securities and Exchange Commission) is a crucial U.S. government agency tasked with protecting investors, maintaining fair and efficient markets, and facilitating capital formation. The SEC enforces federal securities laws and regulates securities exchanges, brokers, and dealers.

I’m building an API that enhances accessibility to SEC filings data. It offers endpoints for querying and retrieving CIK information, with features for data filtering, searching, and integration with financial analysis tools. I’m designing the API to handle large datasets and adhere to SEC data standards.

Technical Details

  • Full-Stack Architecture: I’m employing a full-stack approach using AWS services. The backend is developed with Python and Flask, while the frontend interface supports user interaction. AWS Lambda is used for serverless processing.
  • Data Management: The API efficiently handles extensive SEC data, including features for data filtering, querying, and aggregation. I’m using AWS RDS for data storage and retrieval.
  • Security and Compliance: I’m implementing robust authentication and authorization mechanisms to ensure data security and compliance with SEC standards.
  • Integration: The API integrates with financial analysis platforms, providing valuable insights and supporting data-driven decision-making.

Development and Deployment

In developing this project, I set up the AWS infrastructure, including EC2 instances, RDS databases, and Lambda functions. I’m establishing continuous integration and deployment (CI/CD) pipelines using AWS CodePipeline and CodeDeploy to automate and streamline the deployment process.
